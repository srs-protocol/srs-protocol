概述

OraSRS (Open & Advisory Risk Scoring Service) 是一个咨询式风险评分服务，旨在为互联网安全决策提供权威参考。与传统的阻断式防火墙不同，OraSRS 提供风险评估和建议，由客户端自主决定是否执行相应措施。

设计原则

咨询式服务模式

错误设计: OraSRS 返回 { action: "BLOCK" }

正确设计: OraSRS 返回 { risk_score: 0.92, evidence: ["ddos_bot", "scan_24h"] }

客户端强制执行 → 客户端自主决策是否拦截

类比：OraSRS 是信用评分机构（如 FICO），不是法院。客户端（如银行）自己决定是否采取行动。

内置多重保护机制

  • 分级响应策略：不对所有服务一刀切，关键服务（如医疗、政府）默认放行
  • 自动衰减与申诉通道：风险分随时间衰减（如 24 小时后降级）
  • 透明化与可审计：所有标记记录上链（或公开日志）

公共服务豁免原则

在 OraSRS 策略中硬编码关键公共服务白名单，永不拦截：

  • 政府：.gov, .mil, 国家税务/社保 IP 段
  • 医疗：医院官网、急救系统
  • 金融基础设施：SWIFT、央行支付系统
  • 基础通信：DNS 根服务器、NTP 池

核心功能

风险评估

提供详细的IP和域名风险评分，包含风险等级、证据来源和置信度

关键服务豁免

自动豁免政府、医疗、金融基础设施等公共服务，防止误伤

申诉机制

提供公开申诉接口，允许用户在被误标记时提交申诉

透明化

提供决策依据查询接口，让用户了解被标记的原因

合规性与隐私保护

  • 严格遵循GDPR/CCPA隐私法规
  • 仅存储IP哈希而非原始IP
  • 提供数据删除接口
  • 不直接阻断流量，仅提供建议

实现特点

  • 去中心化：基于节点联邦学习的威胁情报
  • 隐私优先：差分隐私保护，本地数据处理
  • 开源可验证：完全开源，全球审计
  • 标准兼容：支持STIX/TAXII、RPZ等开放标准

新增功能

  • 联邦学习：支持多节点威胁情报安全聚合，保护各节点数据隐私
  • API认证与速率限制：提供API密钥管理和速率限制机制，防止滥用
  • 全面监控：集成Prometheus指标收集和Grafana可视化
  • 容器化部署：支持Docker和Docker Compose一键部署
  • CI/CD流水线：完整的持续集成和持续部署自动化
